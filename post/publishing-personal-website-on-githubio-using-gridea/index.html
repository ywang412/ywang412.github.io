Publishing a personal programming blog on Github.io using Gridea

· 2017-09-20 · # Gadget

I've always wanted to start a programmer's blog because I learned a huge amount from others' tech blog. But hosting and maintaining a blog website could require lots of effort. I came across this Gridea tool (https://github.com/getgridea/gridea) which features html/css static content generation and github.io auto publishing. It helps immensely writing a programming blog.

After downloading and installing the Gridea tool, the first thing is to link it in order to publish on github.io. The settings tab is quite self-explanatory.

The publishing will require a github token for authentication which can be generated here https://github.com/settings/tokens . Remember to use repo scope access token to reduce vulnerability.

The best thing about Gridea is auto-generation of markdown tags and corresponding hotkeys. 

object SumOfMultiples {
  def sum(factors: Set[Int], limit: Int): Int = {
    factors.map(x => Stream.range(1, limit / x + 1).map(_ * x).takeWhile(_ < limit).toList).flatMap(x => x).sum
  }
}

The last thing I want to mention is the comment function. Currently, only Gitalk and Disqus third-party API are allowed. I choose Disqus for its simpler setting.
Register a free account on disqus and create a new application. The public api key will appear in the application confirmation page. The short name can be located on the admin page.

I enjoyed very much the writing and seeing the outcome on github.io. Now I have a personal blog that motivates me to write once a while to share knowledge and help others. Kudos!
