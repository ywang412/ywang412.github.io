LeetCode 929. Unique Email Addresses String.replace() vs String.replaceAll()

· 2019-06-20 · # LeetCode

LeetCode 929 requires removing of "." in a string. It turns out "." and "*" are metaCharacters used in a regex. From JDK 7 manual, the replaceAll() takes regex input and therefore "." was recognized as "any character". Two backward slashes will escape dot character. It is fine to use replace() API which will treat the "." as a dot character.

JDK7

String	replace(char oldChar, char newChar)
Returns a new string resulting from replacing all occurrences of oldChar in this string with newChar.
String	replaceAll(String regex, String replacement)
Replaces each substring of this string that matches the given regular expression with the given replacement.
String	replaceFirst(String regex, String replacement)
Replaces the first substring of this string that matches the given regular expression with the given replacement.

The solution is below,

class Solution {
    public static int numUniqueEmails(String[] emails) {
        if (emails.length < 1) return 0;

        Set<String> set = new HashSet<>();
        for (String email : emails) {
            String[] adds = email.split("@");
            String name = adds[0]; 
            if (name.indexOf("+") >= 0) name = name.substring(0, name.indexOf("+"));
            name = name.replaceAll("\\.","");
            set.add(name + "@" + adds[1]);
        }
        return set.size();
    }

    public static void main(String[] args) {
        String[] t = new String[]{"testemail@leetcode.com","testemail1@leetcode.com","testemail+david@lee.tcode.com"};
        System.out.println(numUniqueEmails(t));
    }
}
